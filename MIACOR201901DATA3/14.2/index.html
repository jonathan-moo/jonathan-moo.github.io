01-Ins_forEach

We introduce a concept called "callback functions" here.

This is given in your notes:

For Loop

									
// Array of student names
var students = ["Johnny", "Tyler", "Bodhi", "Pappas"];

// This function will be called for each element in the array
function printName(name) {
  console.log(name);
}

// Loop through each student name and call the printName function
for (var i = 0; i < students.length; i++) {
  printName(students[i]);

  students[i] = students[i] + " duh";
}

Foreach Loop

									
// Array of student names
var students = ["Johnny", "Tyler", "Bodhi", "Pappas"];

students.forEach(printName);

// You can also define an anonymous function inline
students.forEach(function(name) {
  console.log(name + " duh");

  return name + " duh";
});

If you were to do a console.log(students); for each block, what is the result?

  • For Loop:
    • Will it print "studentName" or "studentName duh"?
  • ForEach Loop:
    • Will it print "studentName" or "studentName duh"?

This is the reason why people complain Javascript is difficult.

There are people who complains it is difficult, but you need to understand why and not join the bandwagon.

05-Evr_Map: Everyone, open up the activity and we'll do it together.

Map

									
var theStagesOfJS = ["confidence", "sadness", "confusion", "realization", "debugging", "satisfaction"];

// Using the .map method
var mapSimpleArray = theStagesOfJS.map(function(item) {
  return item;
});

console.log(mapSimpleArray);

Foreach Loop

									
var forEachStages = theStagesOfJS.forEach(function(each, index) {
  // the return of forEach is ignored
  
  // This will print on your browser
  console.log("ForEach Stages Start:")
  console.log(`Stage ${index + 1}: ${each}`);
  console.log("ForEach Stages End")
  
  return `Stage ${index + 1}: ${each}`;
});

console.log(forEachStages);

Map

  • Returns an array.
  • Creates a new array by executing a function on each element.
  • Use this if you want a new object for each element.

For Each

  • Does not return an array.
  • Does not create a new array when executing a function on each element.
  • Use this if you don't want to change your data accidentally.

06-Evr_Arrow_Functions: Let's open up this activity together.

Some history:

In 2015, some folks decided to revamp Javascript and modernize it. We call this ES6 (or ECMA 2015). Their naming convention needs help because they confuse themselves and us.

The majority of what you've learned in old-school Javascript. It doesn't matter what era you're in. If you only knew Javascript since 1995 and haven't upgraded yourself, you could still work with Javascript (but it's painful).

Then the guys got fancy, and they introduce something called "arrow functions".

Old School

									
var theStagesOfJS = ["confidence", "sadness", "confusion", "realization", "debugging", "satisfaction"];

// Using the .map method
var mapSimpleArray = theStagesOfJS.map(function(item) {
  return item;
});

console.log(mapSimpleArray);

New School

									
var theStagesOfJS = ["confidence", "sadness", "confusion", "realization", "debugging", "satisfaction"];

// Using the .map method
var mapSimpleArray = theStagesOfJS.map((item) => {
  return item;
});

console.log(mapSimpleArray);

This works for EVERY FUNCTION. Just omit the "function", add an arrow, and it will work.

Why didn't we learn this from the start?

Don't ask me. It's part of your curriculum to know how we transit from old-school to modern day Javascript I guess.

07-Ins_Object_Iteration

Remember:

  • Object.keys, Object.values, Object.entries return arrays.
  • Old school code has a habit of building on top of another, and you can you dot notation to chain functions: Object.entries(userInfo).forEach(([key, value]) => console.log(`Key: ${key} and Value ${value}`));
  • Divide & Conquer. Know what it is first before chaining another function to an existing function.
  • forEach work on arrays, not objects directly. We use Object's in-built methods to convert them into arrays, before using forEach to iterate.
