Why Learn?

Tableau is an enterprise visualization software. It takes most data sources and creates visualizations without the need for coding. Most business or data analyst would have some experience in this software.

In a nutshell, Tableau is very good with numbers. Financial reports and metrics may work well with Tableau. But for anything else, you'll need a custom dashboard.

Tableau assumes the data to be CLEAN. It is able to do filtering and stuff, but you'll have to make sure the source data is clean overall.

02-Ins-Groups_Sets

We'll be grouping multiple members in a field into a group for aggregated results.

Groups

For example:

  • Drag State and Postal Code into Rows.
  • Filter the results to United States.
  • Drag Profit into Text under Marks. The sum of the profits will appear according to the state and its corresponding postal code.

Let's say you want to group two fields as a single entity. To do this, select two postal codes (hold Ctrl or Cmd), right click on them and choose Group. If you group two entities together, their profits aggregate as they combine.

Perhaps you want to uniquely identify the group which you've made. You can edit its alias by right-click and choose "Edit Alias".

By grouping, you can selectively identify segments which you want and focus on their results for your visualizations.

Sets

Sets allow us to group dimensions, and create a new column for the dataset. For example, we want to select sub-categories and place them under "Electronics".

  • Drag Sub-Category into Rows, and Measure Names into Columns.
  • Select categories (by holding Ctrl or Cmd) which are electronics (Appliances, Copiers, Machines and Phones), and a mini menu will pop up.
  • In the mini menu, one of the tabs have a symbol with two circles, and if you hover over it, it will have a tooltip which indicates "Create Set". Click on it.
  • Name your set as "Electronics". After you're done, on the bottom left of the screen, your "Electronic" sets will appear as a dimension for you to use.
  • Drag that "Electronics" set into Rows, and a new column will appear.
    • IN indicates that the sub-category is within the "Electronics", while OUT indicates otherwise.

There are also other ways to create sets.

  • Under "Dimensions" on the left panel, right-click "Sub-category" and hover over "Create". It will open a sub-menu and you can see "Set" from there.
  • Instead of selecting sub-categories, this time we'll create a set using conditionals. Select "Condition" on the tab when you create a "Set", and select the "By field" radio button.
  • Perhaps you want to segregate high profits from normal profits if the sum of the profits is above 50k. Do that with the conditionals and label it "High Profits".
  • Do the same with "Low Shipping" if the sum of the shipping cost is below 55k.
  • Drag both sets into Rows, and Tableau will tell you which sub categories give you high profits and low shipping.

We can also combine sets to improve our results. This works so that we can modularize our work, and use some of our sets repeatedly for other things.

  • Right click any of the sets you want to create a combined set with on the left side panel. "Create Combined Set" will pop up as an option.
  • Click on it, and you will be able to select two sets and combine with each other. Tableau will allow you to use joins to indicate what kind of conditions you want, but realistically, you want inner joins. Create "High Profits & Low Shipping" set.
  • Drag the new set into Rows and see the results.

Your sets become a new dimension, and you can use that to augment your graphs just like any other dimensions.

04-Ins-Calculations

Calculations are like macros in Excel. It allows you to put in custom statements such as if and else.

Conditionals

  • Open up GlobalSuperstoreOrders2016.xls.
  • In one of the sheets, on the nav bar at the very top, look for Analysis > Create Calculated Field.
  • Put in your conditionals and it will create a field (Profit Color) for that.
    • IF SUM([Profit]) > 50000 THEN "Very Profitable" ELSEIF SUM([Profit]) < 0 THEN "Unprofitable" ELSE "Not very profitable" END
  • Drag Sub-category into Columns and Profit into Rows.
  • Drag the newly created "Profit Color" field into Color, and Tableau will color code your visuals automatically according to your conditionals.
  • Tableau can deal with unknown quantities. Use this code instead:
    • IIF(SUM([Profit]) > 50000, "Profit", "Loss", "Unknown")
    • If quantity cannot be evaluated, let's say the data isn't clean and it isn't numeric, it will be labeled "Unknown".

Count

Counting the number of items in a column can be valuable, and Tableau offers this function.

  • Drag Measured Names into Columns, and Category into Rows.
  • Drag Measure Values and Number Of Records into Text.
    • As previously taught, "Number Of Records" count the number of rows in the dataset.
  • You can also create another calculated field, and use the function COUNT().
  • To count distinct values, use COUNTD().
  • Drag Category into Columns, and the calculated fields into Rows. You can compare the values of count distinct versus count.

Difference

Tableau allows you to create visualizations by differences.

To create a bar chart based on the changes between each day:

  • Drag Order Date to Columns and change it to Day, and drag 2 Sales pills into Rows. You should get a bar chart.
  • Drag Category into Color to see the categories stacked against each other.
  • Select the right Sales pill and right-click on it. Select Quick Table Calculation > Difference.
  • Hover over one of the bars within the "Difference" chart and you'll see the difference in numbers between your current selection and the previous.
  • By default, Tableau compares against the previous day, but you can change it when you right-click the relevant pill and select Relative to.

11-Ins-LOD: Level Of Detail Calculations

We can introduce more data beneath the current level of visualization. Let's explain this phenonmenon with a map graph.

  • Drag Latitude and Longitude to Rows and Columns respectively.
  • Filter for United States only.
  • Drag State into the Marks pane.
  • We want to create a sum of profits at the national level. However, the current level of visualization is at the state level (since we selected state), we can use a "Level Of Detail" (LOD) calculation to exclude state from the state summation. THis means, we are not summing the results for each state, but for the whole country.
    • Create calculated field for LOD calculation with this code and label it "National Profit":
      • {EXCLUDE [State] : SUM([Profit])}
    • This exclude [State] from the level of detail. We can display on state on the surface and Tableau segregates the results into states automatically, but if we want to overwrite Tableau's innate calculations and aggregate all the state's data, we'll need to exclude [State].
    • Aggregation versus Granulation
      • Aggregration means a higher, more abstract level, while Granulation means a lower, specific level. In our case, state is more granular than national level.
  • Create another calculated field called "Contribution to National Profit, State" with the following code:
    • SUM([Profit]) / ATTR([National Profit]) * 100
    • This gives the percentage of profits relative to the national profit.
    • We created National Profit earlier with our calculated field.
  • Drag "Contribution to National Profit, State" to Color and to Label, and National Profit to Tooltip.
  • ATTR means attribute. An attribute MUST be consistent to all rows since this trait is part-and-parcel of the data. If it's not consistent, ATTR will return an asterisk.
    • Does all states share National Profit? Then it's an attribute.
    • Does all states share the same state profit? No. so it's not an attribute.
