Why We Learn?

D3 means "Data-Driven Documents". It's not only plotting nice graphs and make awesome visualization. The main purpose of D3 is about manipulating DOM (Document Object Model).

You might ask, what is DOM?

You've been dealing HTML for some time, and DOM is simply represented by the tags which you've been using. We call the tags within a HTML as "DOM elements".

The power to manipulate the tags or DOM elements without reloading the page is the reason why we can put awesome visualizations on the web.

This is for Data Visulization Engineers, which is a very specialized field.

01-Evr_ES6

You've been coding with var for very long. It is time to convert var to let or const.

Why?

Javascript was meant to be simple for browsers, but over the years, complexity crept it and Javascript was forced to evolve to suit today's needs. In 1995, var was consistently used. But today, we must use only let or const.

Consider the code here:

							
function logger() {
// returns "undefined", but code still runs.
  console.log(x);
  var x = "hi";
}
logger();

function logger2() {
// ReferenceError: y is not defined. Code will stop execution.
  console.log(y);
  let y = "hello";
}

var is always global in scope. This means, within the code where it is run, the variables are accessible and exposed everywhere. let follows more conventional programming practices, where code within the block can only access the variable in which it was declared.

var allows the variable to be declared but not initialized, but let is declared and initialized at the same time.

From now on, use let instead of var.

What about const?

const is useful if you think that the variable should be immutable, or it should not be changed for anyone else. Things such as Pi, or standards should be defined with const.

But if the variable is an object or array, it will not work as intended!

							
								
const myPets = ["dog", "cat", "rabbit", "some endangered species of sea turtle"];

// myPets = "ferret"; //This will not work - stops execution

// myPets = ["wolf", "giraffe", "parrot"]; //This will not work either

// HOWEVER, we can still manipulate Objects and Arrays!
console.log("before: ", myPets);
myPets.pop();
console.log("after: ", myPets);

04-Ins_Multi_Trace

We want to do multiple trace in a plot:

							
// 04-plot-part

 function randomNumbersBetween0and9(n) {
  var randomNumberArray = [];
  for (var i = 0; i < n; i++) {
    randomNumberArray.push(Math.floor(Math.random() * 10));
  }
  return randomNumberArray;
}

// Create our first trace
var trace1 = {
  x: [1, 2, 3, 4, 5],
  y: randomNumbersBetween0and9(5),
  type: "scatter"
};

  x: [1, 2, 3, 4, 5],
  y: randomNumbersBetween0and9(5),
  type: "scatter"

var data = [trace1, trace2];

// Note that we omitted the layout object this time
// This will use default parameters for the layout
Plotly.newPlot("04-plot-part", data);

06-Ins_Sort_Slice

So you've grabbed a bunch of data from your MongoDB and/or your MySQL. Your data which you received in Javascript seems messy. What happens?

// Array of names
const names = ["Jane", "John", "Jimbo", "Jedediah"];

// Slices first two names
const left = names.slice(0, 2);
console.log(left);

// Slices last two names and return
const right = names.slice(2, 4);
console.log(right);

Sorting: Organization

Sorting is like filters. It goes through the elements one by one.

If the compare function returns a negative number for a pair of numbers, it will put them in the reverse order in the final array: [b, a]

If the compare function returns a positive number for a pair of numbers, it will put them in the input order in the final array: [a, b]

							
// Sorts descending
[3, 2, -120].sort(function compareFunction(firstNum, secondNum) {
   // resulting order is (3, 2, -120)
  return secondNum - firstNum;

  // 3 - firstNum, 2 - secondNum, first run: 2 - 3 < 0, since it gives a negative, [3] is the first element and [2] is placed AFTER [3], which gives [3,2]
  // 2 - firstNum, -120 - secondNum, second run: (-120) - 2 < 0, since it gives a negative, [2] is the first element and [-120] is placed AFTER [2], which gives [2,-120]
  // result = [3,2,-120]
});


// Sorts ascending
[3, 2, -120].sort(function compareFunction(firstNum, secondNum) {
  // resulting order is (-120, 2, 3)
  return firstNum - secondNum;

  // 3 - firstNum, 2 - secondNum, first run: 3 - 2 > 0, since it gives a positive, [3] is the first element and [2] is placed BEFORE [3], which give [2,3]
  // 2 - firstNum, -120 - secondNum, second run: 2 - (-120) > 0, since it gives a positive, [2] is the first element, and [-120] is placed BEFORE [2], which give [-120, 2]
  // result: [-120, 2, 3]
});

// Arrow Function
[3, 2, -120].sort((first, second) => first - second);

Honestly, just remember the syntax above and forget about understanding the whole thing raw. Reverse the firstNum and secondNum to get your ascending or descending.

Also, sorting is in place. This means the original array is being changed. Create a copy first before doing a sort.

Actually, get your MongoDB or MySQL to sort for you when you pull your data. Saves time and effort.
