What is a MVC model?

It stands for Model-View-Controller:

  • Model - Code for business logic
    • Scraping of websites and storing data into DBs
  • View - Code for interaction and interface
    • HTML
  • Controller - Bridge between the model and view, and helps to manage code.
    • Flask Framework

Why do we use it for?

Many engineers call this MVC term differently, but the concept remains the same.

This system of doing things help us to make sure our code is:

  • Reusable
  • Small for other people to work on and others can join you in coding.
  • Eases error checking.
